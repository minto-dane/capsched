# P5A-R2 Selector Model Gate

This model records the pre-code gate after the `0012` boundary review.

It does not approve another Linux behavior patch. The gate only passes when a
future selector direction is modeled as picker-visible before CFS selection,
derived from frozen task-local authority, invalidated on every relevant
generation/epoch/budget/affinity/migration/group event, and constrained by the
future outer Domain/SchedContext selector.

The unsafe configurations reject post-filter production design, unbounded
scans, synthetic authority, pick-time policy lookup, boolean-only subtree
summaries, missing EEVDF-compatible min-summary semantics, missing
group/current semantics, missing fail-closed settlement, unmodeled cross-paths,
hot-layout approval, and runtime/protection/cost/datacenter claims.
