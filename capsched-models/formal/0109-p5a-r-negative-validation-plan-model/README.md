# P5A-R Negative Validation Plan Model

This model backs the P5A-R negative validation plan gate.

It requires the implementation-ready plan to cover late denial, denied execution
publication, same-candidate repick, EEVDF families, group hierarchy, cross-path
exclusion, stale identity, wakeup/newidle leaks, overhead/layout regressions,
and claim overreach before a behavior patch can be considered.
