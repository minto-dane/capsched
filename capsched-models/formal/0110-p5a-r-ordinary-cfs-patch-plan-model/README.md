# P5A-R Ordinary-CFS Patch Plan Model

This model checks the pre-code plan for the first ordinary-CFS-only
SchedExecLease P5A-R behavior patch.

It permits a future Linux behavior patch to be drafted only when the prior
P5A-R gates, ordinary-CFS scope, file boundary, pre-settle picker integration,
hierarchy settlement, cross-path exclusion, bounded attempt-local carrier,
negative validation plan, build/object/QEMU validation matrix, drift evidence,
and security review are required.

It intentionally does not approve runtime denial, CFS deny-and-repick
correctness, production protection, monitor verification, cost-efficiency, or
datacenter readiness.
